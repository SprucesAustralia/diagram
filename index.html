Project Brief: AI-Driven Automation for Spruces Cleaning Services

Overview

Spruces, a commercial cleaning service company, aims to streamline its customer interactions by leveraging AI agents to automate inquiries, quote generation, and booking processes. This capstone project, undertaken by a team from the University of Southern Queensland, focuses on developing a system that enables customers to interact with Spruces through natural language via multiple channels—phone calls, emails, WhatsApp, and website forms—eliminating the need for traditional user interfaces (UIs) where possible. The system will integrate seamlessly with Spruces’ existing Superbase database to manage customer data, cleaner information, and bookings, prioritizing speed and convenience to enhance customer satisfaction and business efficiency.

Objectives

  • Automate Customer Interactions: Develop AI agents to handle customer inquiries, generate instant quotes, and facilitate bookings using voice and text-based communication.
  • Database Integration: Connect all customer interaction data to the Superbase database for storage, retrieval, and management.
  • Enhance Speed and Convenience: Deliver quotes to customers within minutes of their inquiry, minimizing delays and human intervention.
  • Reduce UI Dependency: Shift interactions to natural language channels (voice, text, email) to simplify the customer experience.

Scope

Customer Interaction Module:

  • Voice AI agent (leveraging Cflow or a similar tool) for phone-based inquiries and responses.
  • Text-based AI agent for processing emails and WhatsApp messages.
  • Automated quote generation based on customer-provided details (e.g., facility type, size, cleaning requirements).

Database Integration:

  • Store all interaction data (e.g., customer details, inquiry transcripts, quotes) in Superbase.
  • Enable real-time data retrieval and updates for customer and cleaner management.

Automation Workflows:

  • Develop a Node.js middleware application to process inputs from various channels, interact with AI models, and manage database operations.
  • Implement webhooks and REST APIs to connect communication channels, AI agents, and Superbase.

Security and Privacy:

  • Implement access controls to protect customer data in Superbase.
  • Ensure compliance with privacy standards to prevent unauthorized data access or breaches.

Deliverables

  • Example Use Cases: Customer calls to request a quote, AI processes the request and stores data, quote is sent via email or text.

System Design:

  • Architecture Diagram: Visual representation of system components and data flow (see below).
  • Data Flow Diagrams: Illustrate how data moves from communication channels to middleware, AI agents, and the database.

Implementation:

  • AI Agents: Voice AI (Cflow-based) and text AI for customer interactions.
  • Node.js Middleware: Application with REST APIs and webhooks to handle data processing and integration.
  • Database Integration: Scripts to connect Node.js with Superbase for data storage and retrieval.

Testing:

  • Unit Tests: Validate individual components (e.g., AI response accuracy, API functionality).
  • Integration Tests: Ensure seamless operation between channels, middleware, AI, and database.
  • User Acceptance Testing: Simulate customer interactions to verify system performance and quote delivery speed.

Documentation:

  • Technical Documentation: System architecture, codebase explanation, and API references.
  • User Guide: Instructions for Spruces staff on system maintenance (minimal UI involvement).

  • AI handles 80% of customer inquiries without human intervention.
  • Quotes delivered to customers within 5 minutes of inquiry.
  • Seamless Superbase integration with no data loss.
  • Positive feedback from Spruces on system speed and customer conversion rates.
